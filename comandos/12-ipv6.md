# IPv6 con nftables

## Familias de direcciones

| Familia | Descripción |
|---------|-------------|
| `ip` | Solo IPv4 |
| `ip6` | Solo IPv6 |
| `inet` | IPv4 + IPv6 (recomendado) |

---

## Tabla y cadena para IPv6

```bash
# Tabla exclusiva IPv6
sudo nft add table ip6 filter
sudo nft add chain ip6 filter input '{ type filter hook input priority 0; policy drop; }'

# O usar inet (cubre ambos automáticamente)
sudo nft add table inet filter
```

---

## Reglas IPv6

```bash
# Loopback IPv6
sudo nft add rule ip6 filter input iif lo accept

# Permitir ICMPv6 (OBLIGATORIO para que IPv6 funcione correctamente)
sudo nft add rule ip6 filter input icmpv6 type { nd-neighbor-solicit, nd-neighbor-advert, nd-router-advert } accept
sudo nft add rule ip6 filter input icmpv6 accept

# SSH en IPv6
sudo nft add rule ip6 filter input tcp dport 22 accept

# Bloquear dirección IPv6 específica
sudo nft add rule ip6 filter input ip6 saddr 2001:db8::1 drop

# Permitir subred IPv6
sudo nft add rule ip6 filter input ip6 saddr 2001:db8::/32 accept
```

---

## Reglas inet (IPv4 + IPv6 simultáneamente)

```bash
# Una sola regla que aplica a ambos protocolos
sudo nft add rule inet filter input tcp dport 443 accept

# ICMPv6 dentro de tabla inet
sudo nft add rule inet filter input meta l4proto ipv6-icmp accept

# Conntrack aplica igual para IPv6
sudo nft add rule inet filter input ct state established,related accept
```

---

## Tipos ICMPv6 importantes

| Tipo | Descripción |
|------|-------------|
| `echo-request` | Ping IPv6 |
| `echo-reply` | Respuesta ping |
| `nd-router-advert` | Router Advertisement (RA) |
| `nd-neighbor-solicit` | Neighbor Solicitation (NS) |
| `nd-neighbor-advert` | Neighbor Advertisement (NA) |

```bash
# Permitir solo tipos ICMPv6 necesarios
sudo nft add rule ip6 filter input icmpv6 type { echo-request, nd-neighbor-solicit, nd-neighbor-advert, nd-router-advert } accept
```
