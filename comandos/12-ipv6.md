# IPv6 con nftables

## Diferencia entre familias para IPv4 e IPv6

| Familia | Lo que cubre |
|---------|--------------|
| `ip` | Solo IPv4 |
| `ip6` | Solo IPv6 |
| `inet` | IPv4 e IPv6 al mismo tiempo, el mas recomendado |

Si se usa `inet` para crear la tabla, las reglas aplican a ambos protocolos
y no hay que escribirlas dos veces.

---

## Crear tabla y cadena solo para IPv6

```bash
# Tabla exclusiva para IPv6
sudo nft add table ip6 filter

# Cadena input con politica DROP para IPv6
sudo nft add chain ip6 filter input '{ type filter hook input priority 0; policy drop; }'
```

---

## Reglas basicas para IPv6

### Permitir loopback

```bash
sudo nft add rule ip6 filter input iif lo accept
```

Igual que en IPv4, el loopback siempre debe estar permitido.

### Permitir ICMPv6 - esto es obligatorio para que IPv6 funcione

```bash
# Permitir los mensajes de descubrimiento de vecinos (necesarios para que IPv6 funcione)
sudo nft add rule ip6 filter input icmpv6 type { nd-neighbor-solicit, nd-neighbor-advert, nd-router-advert } accept

# O permitir todo ICMPv6 si se quiere simplificar
sudo nft add rule ip6 filter input icmpv6 accept
```

Sin estos mensajes ICMPv6 el equipo no puede comunicarse en redes IPv6.
Es como el ARP de IPv4 pero para IPv6. Bloquearlos rompe la conectividad.

### Bloquear una IP IPv6 especifica

```bash
sudo nft add rule ip6 filter input ip6 saddr 2001:db8::1 drop
```

`ip6 saddr` es la forma de referirse a la IP de origen en IPv6.

### Permitir una subred IPv6 completa

```bash
sudo nft add rule ip6 filter input ip6 saddr 2001:db8::/32 accept
```

---

## Usar inet para manejar IPv4 e IPv6 juntos

Si la tabla usa `inet`, una sola regla cubre ambos protocolos:

```bash
# Crea la tabla inet (ya cubre los dos)
sudo nft add table inet filter

# Esta regla aplica a IPv4 y IPv6 al mismo tiempo
sudo nft add rule inet filter input tcp dport 443 accept

# ICMPv6 dentro de una tabla inet
sudo nft add rule inet filter input meta l4proto ipv6-icmp accept

# Conntrack funciona igual para IPv6
sudo nft add rule inet filter input ct state established,related accept
```

---

## Tipos de ICMPv6 que hay que conocer

| Tipo | Para que sirve |
|------|----------------|
| `echo-request` | Ping en IPv6 |
| `echo-reply` | Respuesta al ping |
| `nd-router-advert` | Aviso de router en la red |
| `nd-neighbor-solicit` | Buscar la MAC de una IP (como ARP) |
| `nd-neighbor-advert` | Respuesta a la busqueda de MAC |

```bash
# Permitir solo los tipos ICMPv6 necesarios para que la red funcione
sudo nft add rule ip6 filter input icmpv6 type { echo-request, nd-neighbor-solicit, nd-neighbor-advert, nd-router-advert } accept
```
